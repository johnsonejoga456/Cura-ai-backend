5. DEPLOYMENT STRUCTURE
================================================================================

Production Environment:

BACKEND (Railway/Render):
  cura-ai-backend/ (deployed as containerized service)
  ├── Environment: Production
  ├── Domain: api.cura-ai.com
  └── Services: PostgreSQL, Redis

WEB (Vercel):
  cura-ai-web/ (deployed to Vercel)
  ├── Environment: Production
  ├── Domain: app.cura-ai.com
  └── CDN: Vercel Edge Network

MOBILE (App Stores):
  cura-ai-mobile/ (built and deployed)
  ├── iOS: App Store
  └── Android: Google Play Store

6. FILE NAMING CONVENTIONS
================================================================================

BACKEND (Python):
  - snake_case for files: auth_service.py, user_model.py
  - Classes: PascalCase (UserCreate, AuthService)
  - Functions: snake_case (get_current_user, create_token)
  - Constants: UPPER_SNAKE_CASE (API_VERSION, MAX_FILE_SIZE)

MOBILE (Flutter/Dart):
  - snake_case for files: auth_provider.dart, chat_screen.dart
  - Classes: PascalCase (AuthProvider, ChatScreen)
  - Variables: camelCase (userName, isLoading)
  - Constants: lowerCamelCase or SCREAMING_SNAKE_CASE

WEB (TypeScript):
  - kebab-case for files: auth-service.ts, chat-window.tsx
  - Components: PascalCase (ChatWindow, MessageBubble)
  - Functions: camelCase (sendMessage, getUserProfile)
  - Constants: UPPER_SNAKE_CASE (API_URL, MAX_FILE_SIZE)


================================================================================
7. IMPORTANT NOTES
================================================================================

1. ENVIRONMENT FILES:
   Never commit .env files to version control!
   Always use .env.example as a template.

2. SECRET KEYS:
   Store sensitive data (API keys, tokens) in environment variables.

3. FIREBASE CONFIG:
   Keep google-services.json and GoogleService-Info.plist out of git.

4. NODE_MODULES / VENV:
   Always gitignore dependency directories.

5. BUILD ARTIFACTS:
   Gitignore build outputs (dist/, build/, .next/, etc.)
